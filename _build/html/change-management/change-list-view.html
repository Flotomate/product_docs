Flotomate provides an intuitive interface to manage the processes associated with Change Management. The entire interface is divided into Change List View and Change Details View.

The Change List View is the central hub for Technicians to track and manage all Changes in the tool.

To open Change List View

  1. Log in to your Dashboard from the Technician Portal.
  2. Click on Change from the Navigation Tabs. The Change List View opens with the default filter set as All Open Changes.

Please refer to Fig. 208 for the below description:

  • Section-A, B, & C are the search features.

  • Section-D houses the bulk-operations which are:

    1. Bulk Update
    2. Archive
    3. Claim and Assign

  • Section-E is the list area for all the Changes in the system. Visibility of a Change depends on the applied filters, and the way Changes are shown depends on whether Detailed View is selected or Glance View (section-G in Fig. 208). Regardless of the view, the following information is visible about a Change:

    1. Three classifiers which are:

      1. The stage of the Change as Status.
      2. The Priority of the Change which can be modified.
      3. The Due status of the Change which can be modified by click on the label.

    2. Name of the requester and create time of the Change.

    3. The subject line of the Change along with its ID.

    4. Assignee information or the Claim and Assign button. If the Change is closed, then you see the Reopen button.

    5. Action menu next to Assign or Reopen houses the View Audit Trail and Open in New Tab options.

  • Section-F is the Glance Pane (visible only in Grid View)of a Change, and it shows the following information:

    1. Change ID and Subject line.
    2. The entire Metadata of the Change.
    3. Requester details under Details tab.
    4. The Activity tab shows the Audit Trail of the Change.
    5. The Attached CI tab lists all Assets linked to the Change.

  • Section-G is where you select the maximum number of Changes visible per page, and the view, Table View or Grid View.

    1. In Grid View, the listings are more descriptive and explicit compared to Table View. You can view additional information in the Glance Pane of a Changes.
    1. Table View is minimalistic which shows minimum information without the Glance Pane. The focus is to give you the most critical information.
