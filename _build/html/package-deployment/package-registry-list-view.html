Packages/Registry List ViewÂ¶

The UI is divided across packages/registries and Computers. One aspect lets you view packages & registry templates and perform related operations, and the Computer aspect lets you view all Computers and perform related operations.

We have the Package/Registry List View that gives a point & click interface to manage all package/registry from a single place. The view is divided across Windows and Linux platform.

Viewing Package/Registry List View

  1. Log in to your Dashboard.
  2. Click on the Launcher icon and select Package.
  3. Now choose a type: either Windows Packages, Linux Packages or Windows Registry. Based on your selection packages/registries are shown.

Please refer to Fig. 865 for the below description:

  • Section-A houses the search features.

  • Section-B is the list area where you can view all the packages/registries in the system. You can view the following information about a package/registry:

    Packages info:

    1. Package ID as generated by the system to identify the Package.
    2. Name of the Package.
    3. Version of the package.
    4. Package type is a system generated classification which can be either:
      1. Application Patch: Refers to a component of a computer program.
      2. Application: Refers to an entire program.
      3. Operating System: OS like Windows.
      4. OS Component Patch: Update patch for an OS.
      5. Unknown
    5. Package Architecture which is either 64bit or 32bit.

    Registry info:

    1. Name of the Registry template.
    2. Description of the Registry template.
    3. The number of registry items in the template.

  • Section-C houses the option to add a new package or create a new registry template.
